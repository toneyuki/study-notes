# AWS CLIメモ

AWS CLIに関するコマンドや知識をまとめる。

---

## ❗ 前提

AWS CLIはアクセスキーを発行して利用するが、  
**ルートユーザー（AWSに登録しているEmailアドレス等）では発行してはならない。**

🔗 参考資料  
- [AWS アカウントのルートユーザーのベストプラクティス（AWS公式）](https://docs.aws.amazon.com/ja_jp/IAM/latest/UserGuide/root-user-best-practices.html#ru-bp-access)

---

ルートユーザーが乗っ取られた場合、以下のような被害が考えられる。  
そのため、MFA等の多要素認証でのみログインするようにする。

### ⚠️ 想定されるリスク

#### 💸 高額請求
暗号資産マイニングのために高性能インスタンスを乱立させられ、  
数百万～数千万単位で請求される可能性がある。

#### 🔒 アカウントの乗っ取り
メールアドレスやパスワードを変更され、  
利用者本人がログインできなくなる。

#### 🗑️ データの削除・漏洩
S3内部のデータが盗み出され、  
価値のある情報は売却される恐れがある。

#### 🛑 攻撃者としての利用
作成されたインスタンスからサイバー攻撃やスパムが送信され、  
管理者として法的リスクを負う可能性がある。

---

## ✅ 対策

- IAMユーザーを作成する  
- IAMユーザーに対してアクセスキーを発行する  
- 必要最低限の権限（ロール）を付与する  
- IAMユーザーやリソースを監視する  
- 不審な操作があればアクセスキーやIAMユーザーを削除する  

💡 インターネット上では、アクセスキーの漏洩を監視するbotが常に存在するため注意する
